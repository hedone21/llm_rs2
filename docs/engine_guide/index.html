llm.rs Engine Guide

기준 커밋: 46ee4c2d · 작성일: 2026-06-10 · llm.rs Engine Guide

이 가이드에 대해

llm.rs(저장소명 llm_rs2) 추론 엔진의 온보딩 가이드다. 연구실 구성원이 코드 리뷰·구현·개선 작업에 진입할 때 필요한 시스템 지도, 도메인 어휘, 확장 절차를 12개 챕터로 제공한다. 독자는 Rust 중급(문법 설명 없음)과 LLM 추론 기초(attention, KV cache 개념)를 갖췄다고 가정한다. 모든 챕터는 기준 커밋의 코드만 근거로 독립 작성되었다 — 본문의 각 주장에는 실제 코드 심볼 위치(파일::심볼 (L행))가 근거로 달려 있다.

두 프로세스(Engine ⊥ Manager, 결합은 IPC뿐) 위에서 Format(저장 표현) ⊥ Stage(관리 동작) ⊥ Hardware(실행 위치) 3축이 직교한다. 전체 지도는 01장, 축의 정의는 02장 참조.

챕터 목차

01. 시스템 개요

두 프로세스·세 축·토큰당 8단계 루프의 전체 지도

02. 도메인 용어

Format ⊥ Stage ⊥ Hardware 3축과 핵심 enum 어휘 사전

03. 코드베이스 지도

engine/src 41개 모듈 책임표 + 의존 그래프 + 워크스페이스 구조

04. 추론 파이프라인

Prefill/Decode 경로와 GQA·plan 핫패스 해부

05. 핵심 추상화 카탈로그

5대 trait의 시그니처·계약·구현체·설계 동기

06. 메모리 & 데이터 포맷

버퍼 계층·zero-copy 두 갈래·quant 블록 바이트 해부

07. KV 캐시 구조와 관리

KV cache 레이아웃·2-tier 압력 관리·eviction/양자화 내부

08. Resilience & IPC

Manager IPC 신호→명령→엔진 반응 전 경로 해부

09. 백엔드 구현

Backend trait 계약과 CPU/OpenCL/plugin 3경로 디스패치 해부

10. 확장 How-to

핸들러·정책·백엔드·모델 4축 확장 절차와 등록 지점 전수

11. 개발 워크플로우

빌드·크로스컴파일·테스트 계층·성능 측정 함정

12. 컨벤션 & 리뷰 체크리스트

단위·레이아웃·위치 불변식과 PR 리뷰 체크리스트

역할별 읽기 경로

  • 코드 리뷰부터 시작 — 02 → 03 → 05 → 12. 이 코드베이스의 PR 지적 1위는 버그가 아니라 용어·불변식 위반이다 — 02와 12의 안티패턴 표가 리뷰 무기다.
  • 구현·개선 시작 — 02 → 03 → 04 → 해당 영역 챕터(메모리 06 · KV 캐시 07 · IPC 08 · 백엔드 09) → 10 → 11.
