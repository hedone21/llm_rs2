# Round 9: EMR 비선형성 원인 조사 리포트

## 문제 정의

Round 8에서 관찰된 EMR의 비선형 패턴:

| SeqLen | EMR | FDT | 해석 |
|--------|-----|-----|------|
| 1024 | 0.990 | 569 | 높음 |
| 2048 | 0.800 | 1625 | 중간 |
| **4096** | **0.522** | **2139** | **최저** |
| 8192 | 0.939 | 7691 | 높음 |

시퀀스 길이가 증가하면서 EMR이 단조 감소하지 않고, 4096에서 최저 후 8192에서 다시 상승.
**"eviction 품질이 시퀀스 길이에 비선형적으로 의존"하는 것처럼 보이는 현상의 원인은?**

---

## Phase 1: 데이터 정밀 분석

### 발견 1: EMR ≈ FDT/total (재수렴 없음)

| SeqLen | FDT/total | EMR |
|--------|-----------|-----|
| 2048 | 1625/2030 = 0.800 | 0.800 |
| 4096 | 2139/4095 = 0.522 | 0.522 |
| 8192 | 7691/8191 = 0.939 | 0.939 |

첫 발산(FDT) 이후 Post-FDT 재수렴율: **0.0%** (모든 케이스).
EMR은 순수하게 FDT 위치에 의해 결정됨.

### 발견 2: H2O와 Sliding은 완전히 동일

| SeqLen | H2O FDT | Sliding FDT | H2O-SL 토큰 일치 |
|--------|---------|-------------|-------------------|
| 2K | 1625 | 1625 | 91.9% (2K 끝부분만 차이) |
| 4K | 2139 | 2139 | **100.0%** |
| 8K | 7691 | 7691 | **100.0%** |

두 정책이 동일한 FDT, 동일한 eviction 수, 동일한 cache_pos 궤적을 보임.
**→ 비선형성은 eviction 전략 차이가 아님**

### 발견 3: 모델이 ~103토큰 반복 주기로 텍스트 생성

```
B-2048: 50-gram 반복 18회, 간격 [104, 103, 103, ...]
B-4096: 50-gram 반복 38회, 간격 [104, 103, 103, ...]
B-8192: 50-gram 반복 77회, 간격 [104, 103, 103, ...]
```

Baseline이 ~103토큰 주기로 동일한 텍스트를 반복 생성하는 **반복 루프** 상태.

### 발견 4: FDT는 반복 주기 내 "결정 지점"에서 발생

각 반복 주기에 top-1과 top-2 logit 차이가 매우 좁은 지점이 존재:

| SeqLen | FDT | Base 선택 | Exp 선택 | top1-top2 gap |
|--------|-----|-----------|----------|---------------|
| 2K | 1625 | " female" | " one" | 1.89 (좁음) |
| 4K | 2139 | " one" | " female" | 1.75 (좁음) |
| 8K | 7691 | " did" | " used" | 1.26 (매우 좁음) |

Eviction으로 인한 logit 분포 미세 섭동이 이 결정 지점에서 토큰 선택을 뒤집음.

### 발견 5: FDT 갭 = 위상 정렬 문제

| SeqLen | Eviction→FDT 갭 | 해석 |
|--------|-----------------|------|
| 2K | 601 토큰 (6 주기) | 6주기 후 결정 지점에서 flip |
| **4K** | **91 토큰 (1 주기)** | **즉시 다음 결정 지점에서 flip** |
| 8K | 3595 토큰 (35 주기) | 35주기 동안 축적 후 flip |

4K에서 EMR이 최저인 이유: eviction 직후 첫 번째 결정 지점이 91토큰 후에 위치하여
섭동이 즉시 토큰 선택을 뒤집고, 이후 48%의 시퀀스가 다른 반복 패턴으로 진입.

## Phase 2: 프롬프트 다양성 검증

### 실험 설계

3개 프롬프트로 동일 조건(50% eviction, greedy decoding) 실험:

| ID | 프롬프트 | 반복성 |
|----|----------|--------|
| P1 | "The history of artificial intelligence..." | 높음 (~103토큰 주기) |
| P2 | "Chapter 1: Introduction to Computer Networks..." | 낮음 |
| P3 | "Once upon a time in a small village..." | 매우 낮음 |

### 결과

| Prompt | SeqLen | EMR | FDT | FDT gap |
|--------|--------|-----|-----|---------|
| P1 | 2K | 0.833 | 1625 | 601 |
| **P1** | **4K** | **0.522** | **2139** | **91** |
| **P2** | **2K** | **1.000** | **2047** | **∞ (발산 없음)** |
| **P2** | **4K** | **1.000** | **4095** | **∞ (발산 없음)** |
| **P3** | **2K** | **1.000** | **2047** | **∞ (발산 없음)** |
| **P3** | **4K** | **1.000** | **4095** | **∞ (발산 없음)** |

### 해석

**P2, P3 프롬프트에서 50% eviction 후 EMR = 1.000 (완전 일치!)**

- 반복성이 낮은 프롬프트에서는 eviction이 **품질에 전혀 영향을 주지 않음**
- 2K와 4K 모두 단 하나의 토큰도 발산하지 않음
- 이는 모델이 최근 컨텍스트만으로 동일한 토큰을 생성할 수 있음을 의미

**→ Round 8의 비선형 EMR 패턴은 측정 아티팩트이며, eviction 품질의 실제 문제가 아님.**

---

## 근본 원인 요약

```
비선형 EMR 패턴의 원인:
  ① 모델이 ~103토큰 주기로 반복 텍스트 생성 (greedy decoding + 소형 모델)
  ② 반복 주기 내 "결정 지점"에서 top-1/top-2 logit 차이 < 2.0
  ③ Eviction으로 인한 logit 섭동이 결정 지점에서 토큰 선택을 뒤집음
  ④ 뒤집힘 발생 시점은 eviction 위치와 결정 지점의 위상 정렬에 의존
  ⑤ 한 번 발산하면 다른 반복 패턴으로 진입 → 재수렴 불가

결론: 측정 아티팩트 (eviction 전략의 실질적 차이가 아님)
```

## 함의

### 실험 방법론
1. **반복 텍스트로는 eviction 품질을 정확히 측정할 수 없음**
2. **다양한 프롬프트의 앙상블**로 EMR을 평가해야 의미 있는 결과
3. **Greedy decoding은 단일 토큰 flip → 전체 시퀀스 발산**을 유발 → 과도하게 민감한 메트릭

### Eviction 품질
1. **50% 일괄 eviction은 대부분의 컨텐츠에서 품질 손실 없음** (P2, P3: EMR=1.000)
2. **H2O와 Sliding은 대규모 시퀀스에서 동일한 동작** (동일 토큰 보존)
3. **최근 컨텍스트(recent window)가 토큰 생성 품질의 지배적 요인**

### 향후 실험 설계 제안
1. 프롬프트 다양성 확보 (최소 3개 이상의 프롬프트 앙상블)
2. Top-K Overlap 메트릭 활용 (단일 토큰 flip에 강건)
3. Nucleus sampling (top-p)로 약간의 확률성 도입
4. Perplexity 또는 cross-entropy 기반 메트릭 (토큰 정확도보다 분포 유사도)
