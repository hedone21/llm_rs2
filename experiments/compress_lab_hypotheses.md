# F16 KV Cache Compression — 가설 실험 로그

## 데이터셋

| 항목 | 값 |
|------|---|
| 모델 | Llama 3.2 1B |
| 레이어 | 16 |
| KV heads | 8 |
| Head dim | 64 |
| 토큰 | 161 (prompt 33 + decode 128) |
| dtype | F16 |
| Layout | SeqMajor |
| 총 크기 | 5.0 MB |
| 레이어당 K 또는 V 크기 | 161 × 512 × 2 = 164,864 bytes |

## 엔트로피 분석 (Layer 0 K)

| 영역 | bits/byte | 최대 대비 | 여유 |
|------|-----------|----------|------|
| 전체 (F16 바이트) | 7.387 | 92.3% | 7.7% |
| High byte (sign+exp+man[9:8]) | 5.665 | 70.8% | **29.2%** |
| Low byte (man[7:0]) | 7.991 | 99.9% | **0.1%** |

Exponent 분포 (상위 5): exp=15(29.0%), exp=14(21.1%), exp=16(18.5%), exp=13(12.7%), exp=12(6.9%). unique=19/31.

**핵심 관찰**: F16의 16비트 중 가수 하위 8비트(50%)는 엔트로피 7.991로 사실상 비압축성. 압축 가능한 정보는 상위 바이트에 집중.

## 기준선 (Baseline)

데이터: 실제 Llama 3.2 1B KV cache, 16 layers × (K+V), elem_size=2

| # | Algorithm | Ratio | Comp ms | Decomp ms | 비고 |
|---|-----------|-------|---------|-----------|------|
| B1 | raw_lz4 | 1.006x | 0.50 | 0.19 | 거의 효과 없음 |
| B2 | raw_zstd | 1.094x | 4.20 | 3.58 | 엔트로피 코딩만으로 ~9% |
| B3 | byteshuffle+lz4 | 1.009x | 0.87 | 0.39 | byte 분리는 효과 미미 |
| B4 | bitshuffle+lz4 | 1.058x | 15.56 | 16.29 | bit-plane 분리, 느림 |
| B5 | bitshuffle+zstd | 1.076x | 19.24 | 17.06 | bit-plane + 엔트로피 |
| B6 | expsplit+lz4 | 1.009x | 4.78 | 0.33 | hi/lo 분리만으로는 부족 |
| B7 | **expsplit+bytedelta+zstd** | **1.131x** | 6.74 | 3.29 | **현재 최고** |
| B8 | chanreorder+bitshuffle+lz4 | 1.069x | 19.09 | 27.96 | 채널 재배열 효과 미미 |
| B9 | delta_token+bitshuffle+lz4 | 1.044x | 15.62 | 16.40 | 토큰 간 XOR, 오히려 역효과 |

**현재 최고 무손실**: expsplit+bytedelta+zstd = **1.131x** (comp 6.7ms, decomp 3.3ms)

## 가설 실험 결과

### H1: F16 심볼(16비트) 단위 엔트로피 코딩

- **가설**: F16을 바이트가 아닌 16비트 심볼 단위로 엔트로피 코딩하면 더 좋은 압축률을 얻을 수 있다
- **방법**: Layer 0 K 데이터에서 16비트 심볼 엔트로피, unique 값 수, 빈도 분포 측정
- **결과**:

| 측정 | 값 |
|------|---|
| F16 심볼 엔트로피 | 13.394 bits/symbol (max 16.0) |
| 이론적 최대 압축률 | **1.195x** |
| Unique F16 값 | 15,490 / 82,432 total (23.6% of 65,536) |
| 최빈값 | 0x0000 (0.0) = 512회 (0.62%) |
| 2위 이하 | 각 0.03~0.04% — 매우 균등 분포 |

- **판정**: ⚠️ **이론 상한 확인**
  - 이론적으로 **1.195x**가 F16 KV 캐시의 무손실 압축 한계.
  - 현재 최고 1.131x는 이론 한계의 94.6% 달성 → **이미 거의 최적에 근접**.
  - 0.0 값(512개)을 제외하면 값 분포가 극도로 균등. 허프만/ANS로도 큰 개선 불가.
  - **결론: 무손실 F16 압축은 ~1.2x가 물리적 한계.**

### H4: 가수 비트별 엔트로피 프로파일

- **가설**: 가수 상위 비트는 지수와 상관관계가 있어 엔트로피가 낮을 수 있다
- **방법**: 각 비트 위치별 P(1) 및 이진 엔트로피 측정
- **결과**:

| Bit | Region | P(1) | Entropy | 비고 |
|-----|--------|------|---------|------|
| 15 | sign | 0.4812 | 0.999 | 거의 50:50 |
| 14 | exp | 0.2255 | 0.770 | **가장 낮은 엔트로피** |
| 13 | exp | 0.7636 | 0.789 | |
| 12 | exp | 0.7014 | 0.880 | |
| 11 | exp | 0.5647 | 0.988 | |
| 10 | exp | 0.4988 | 1.000 | 거의 랜덤 |
| 9 | man | 0.4076 | 0.975 | 약간의 편향 |
| 8 | man | 0.4533 | 0.994 | |
| 7-0 | man | ~0.49-0.50 | ~1.000 | **완전 랜덤** |

- **판정**: ⚠️ **부분 확인**
  - 지수 bit 14,13만 유의미한 편향 (0.770, 0.789). 나머지는 거의 최대 엔트로피.
  - 가수 bit 9는 약간의 편향(0.975)이 있으나, bit 8 이하는 완전 랜덤.
  - 16비트 중 **실질적으로 압축 가능한 비트는 2~3개뿐** (bit 14, 13, 12).
  - Bitshuffle가 이 비트들을 이미 분리하고 있으므로 추가 개선 여지 적음.

### H2: 지수별 그룹핑 후 가수 엔트로피

- **가설**: 같은 exponent를 가진 값끼리 그룹핑하면 가수 분포가 더 좁아져 압축률이 올라간다
- **방법**: exponent별로 가수(10비트) 분리 → 각 그룹의 심볼 엔트로피 측정
- **결과**:

| Exp | Count | Unique mantissas | Entropy (10b) | Ent/max |
|-----|-------|-----------------|---------------|---------|
| 0 | 514 | 3 | 0.041 | 0.4% |
| 12 | 5,667 | 1,018 | 9.863 | 98.6% |
| 13 | 10,487 | 1,024 | 9.924 | 99.2% |
| 14 | 17,431 | 1,024 | 9.951 | 99.5% |
| 15 | 23,866 | 1,024 | 9.943 | 99.4% |
| 16 | 15,280 | 1,024 | 9.714 | 97.1% |

- **판정**: ❌ **기각**
  - 주요 exponent 그룹(12~16)에서 가수 unique=1024 (10비트 최대), 엔트로피 97~99%.
  - 같은 exponent 내에서도 가수는 **거의 균등 분포** → 그룹핑 효과 없음.
  - exp=0(값 0.0) 그룹만 예외적으로 낮은 엔트로피 (0.4%).

### H3: 인접 채널 간 XOR delta

- **가설**: 같은 토큰 내 인접 채널(head_dim) 값은 비슷하여 XOR delta가 엔트로피를 낮춘다
- **방법**: 인접 F16 값 간 XOR/SUB delta → 엔트로피 비교
- **결과**:

| 방법 | Byte entropy | 원본 대비 |
|------|-------------|----------|
| 원본 | 7.387 | — |
| XOR delta | 7.582 | **+0.195 (악화)** |
| SUB delta | 7.530 | **+0.143 (악화)** |
| XOR F16 entropy | 13.990 | 원본 13.394 대비 악화 |

- **판정**: ❌ **기각**
  - 인접 채널 값은 상관관계가 **없음**. Delta가 오히려 엔트로피를 증가시킴.
  - 토큰 간 delta(B9)도 이미 실패했으므로, **F16 KV 값은 공간적/시간적 상관관계 없음**.

### H8: Zstd 압축 레벨 증가

- **가설**: Zstd level을 올리면 FSE 엔트로피 코더가 더 정교한 모델을 사용하여 압축률 향상
- **방법**: expsplit+bytedelta+zstd 파이프라인에서 level 1/3/5/9 비교
- **결과**:

| Level | Ratio | Comp ms | Decomp ms |
|-------|-------|---------|-----------|
| 1 | 1.131x | 9.13 | 4.68 |
| 3 | 1.129x | 6.90 | 3.17 |
| 5 | 1.131x | 8.72 | 3.07 |
| 9 | 1.131x | 8.55 | 3.13 |

- **판정**: ❌ **개선 없음**
  - Level 1이 이미 최적. 데이터 크기가 작아 Zstd 내부 FSE 테이블이 level에 무관하게 동일 수렴.
  - Level 3이 오히려 약간 나쁨 (LZ77 매칭 전략 차이). 해제 속도는 level 무관하게 일정.

### H9: Hi stream bitshuffle + Zstd

- **가설**: Hi byte를 bitshuffle로 비트별 분리하면 bit 14,13의 편향을 더 효율적으로 잡을 수 있다
- **방법**: expsplit → hi stream bitshuffle(elem=1) → Zstd, lo stream → LZ4
- **결과**: **1.072x** (기준 1.131x 대비 열등)
- **판정**: ❌ **기각**
  - Bytedelta(인접 값 차이)가 bitshuffle(비트별 분리)보다 hi byte의 구조를 더 잘 포착.
  - Hi byte는 8비트 값으로 인접 값 간 연속성이 강함 → bytedelta가 우세.

### H10: Hi↔Lo 상호정보 활용 (조건부 코딩)

- **가설**: Hi와 Lo 바이트 사이의 상호정보(0.261 bits)를 context coding으로 회수
- **방법**: Hi byte 값별로 Lo byte를 별도 버킷에 모아 각각 Zstd 압축
- **분석**:

| 측정 | 값 |
|------|---|
| H(Hi) | 5.665 bits |
| H(Lo) | 7.991 bits |
| H(Hi,Lo) joint | 13.394 bits |
| I(Hi;Lo) | **0.261 bits/symbol** (H(Lo)의 3.3%) |
| 독립 코딩 합 | 13.656 bits |
| 최적 조인트 | 13.394 bits |

- **결과**: **1.117x** (기준 1.131x 대비 열등)
- **판정**: ❌ **기각**
  - 상호정보 0.261 bits는 존재하지만, 버킷 분할 시 각 버킷 크기가 작아져 Zstd 오버헤드(헤더, FSE 테이블)가 이득을 초과.
  - 이론적으로도 0.261 bits 회수는 ~0.02x 개선에 불과.

### H11: 3-stream 비트필드 분리

- **가설**: sign(1b) + exponent(5b) + mantissa(10b)로 비트필드 경계에 맞춰 분리하면 더 효율적
- **방법**: sign → bit-pack, exp → bytedelta+Zstd, mantissa → 2바이트 LE pack+LZ4
- **결과**: **0.790x** (압축 실패 — 원본보다 커짐!)
- **판정**: ❌ **기각**
  - 10비트 mantissa를 16비트(2바이트)로 패킹하면 6비트 낭비 → 데이터가 커짐.
  - 올바른 구현은 10비트 비트스트림 패킹이 필요하나, 복잡도 대비 이론 이득(~0.26 bits)이 미미.

### H12: Head별 독립 압축

- **가설**: KV head별로 값 분포가 달라 독립 압축 시 전체 엔트로피 감소
- **방법**: 8개 head별 데이터 분리 → 각각 expsplit+bytedelta+Zstd
- **분석 — Head별 엔트로피**:

| Head | Overall | Hi byte | Lo byte |
|------|---------|---------|---------|
| 0 | 7.264 | **5.345** | 7.974 |
| 1 | 7.290 | 5.407 | 7.974 |
| 2 | 7.343 | 5.540 | 7.978 |
| 3 | 7.388 | 5.699 | 7.973 |
| 4 | 7.387 | 5.699 | 7.972 |
| 5 | 7.373 | 5.614 | 7.975 |
| 6 | 7.323 | 5.470 | 7.970 |
| 7 | 7.319 | 5.495 | 7.974 |

- **결과**: **1.127x** (기준 1.131x 대비 소폭 열등)
- **판정**: ❌ **기각**
  - Head 0이 가장 낮은 엔트로피(5.345)를 보이나, 분할 시 데이터 크기 감소로 Zstd 효율 저하.
  - 8개 스트림의 헤더 오버헤드가 head별 엔트로피 차이(Δ0.35 bits)보다 큼.

## 종합 결론

| 순위 | Algorithm | Ratio | 이론 한계 대비 |
|------|-----------|-------|--------------|
| 1 | expsplit+bytedelta+zstd | **1.131x** | **94.6%** |
| 2 | raw_zstd | 1.094x | 91.6% |
| 3 | bitshuffle+zstd | 1.076x | 90.0% |
| — | 이론 한계 (H1) | 1.195x | 100% |

**무손실 F16 KV 캐시 압축의 결론**:
- 이론 한계 **1.195x** (13.394 bits/symbol), 현재 최고 **1.131x** (94.6% 달성).
- 남은 갭 0.064x (0.76 bits/symbol)는 모두 lo byte의 근소한 비균등성에 있으나, 이를 활용하는 모든 시도(H9~H12)가 오버헤드 때문에 실패.
- **expsplit+bytedelta+zstd가 사실상의 실용적 최적해**.
- 의미 있는 압축률(2x+)은 **lossy 접근(H7)**으로만 가능.
